# TSA-UTE-Text-Based-Interface

The idea of Autoschedule came as we were looking at how we organize our work in high school to make sure the work gets completed in timely order. Since there is a lot of high school work, organizing it manually takes a lot of time. The work we organize also gets repeated from time to time. To avoid this problem, we decided to tackle this problem for our text-based user interface coding application for TSA. After some analysis, we decided to code Autoschedule by using the idea of organizing works and groups entered by the user. Each group will contain a priority number that will tell us which works need to be put on top of the schedule. After coding this main functionality, we decided tap into the potential of Autoschedule by adding more features for the user to make it easier for them to use Autoschedule. These features include changing the starting time and break time, allowing the user to enter previously entered works, stores the user’s schedule in a text file at the end of the code, saves your schedule and details for the future using files, allows you to make direct changes to the schedule after it has been completed, and has flexibility to counteract accidental and faulty user inputs!

Commands to get started:
1. Needs a Gemini API key to work (put in the .env file).
2. Install python dependencies (pip install -r requirements.txt)
3. python main.py
